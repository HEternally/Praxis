# Aesthetic Vocabulary

design-ui 审美陈述的具体词汇库。SKILL.md 的 Hard constraint 引用本文件。

## 禁用空词清单

这些词没有任何视觉锚点 — 是 LLM 默认输出的占位词,审美陈述里出现就视为没设计:

- 现代 / 简洁 / 简约 / 清爽
- 高级感 / 极简 / 优雅 / 精致
- 大气 / 舒适 / 干净 / 美观
- 漂亮 / 时尚 / 人性化

## 替代:具体审美描述

### 字体
- 衬线 vs 无衬线 vs 等宽
- 字重: 细 100 / 标准 400 / 加粗 700 / 黑 900
- 字高: 1.2 (紧凑) vs 1.5 (标准) vs 1.8 (松散)
- 字号节奏: 14/16/20/28/40 (步长 1.4 vs 1.6 vs 黄金比)

### 间距与密度
- 密度低字号大 (类似 Apple 官网)
- 留白多 + 元素孤立 (类似 Linear / Stripe)
- 元素紧凑信息密集 (类似 Bloomberg / Notion 数据库)
- 网格对齐 8px / 4px

### 颜色
- 高对比纯色 (#000 + #fff)
- 灰阶层级 (3-5 级灰)
- 单色调 + 单点 accent
- 渐变色 vs 纯色

### 形状
- 卡片硬边 (border-radius: 0)
- 半圆角 (4-8px)
- 全圆角 (16px+)
- 透明 / 半透明 (backdrop-blur) / 实心
- 阴影: 无 / 浅 / 多层

### 按钮风格
- ghost (描边) vs 实心 (filled)
- 文字按钮 vs 图标按钮
- 大尺寸感 (padding 大) vs 紧凑

## 整体定位例子

陈述审美时可以这样写,让人立刻有视觉锚点:

- "类 Linear: 极致精简,大量留白,字号大,无装饰边框"
- "类 Notion: 灰白基调,字体无衬线偏圆,信息密度中等,卡片浅阴影"
- "类 Stripe: 单色品牌色 + 渐变 accent,圆角 8px,字体无衬线,信息密度低"
- "类 Bloomberg: 信息密度极高,字号小 (12-14),等宽字体,黑底高饱和强调色,功能优先"
- "类 Apple 官网: 字号阶梯大 (28/40/56),通栏图,中央对齐,留白极多"
- "类 Vercel: 黑白主调 + 单点 cyan,字体几何感强,卡片硬边或微圆角"

## 写法

审美陈述 2-3 句,每句包含至少 1 个具体描述项:

> 整体类 Linear 的极致精简,大量留白让信息呼吸。字体无衬线偏几何,字号阶梯 14/16/20/32 节奏明显。按钮 ghost 而非实心,只有提交动作用单色填充作为 anchor。
