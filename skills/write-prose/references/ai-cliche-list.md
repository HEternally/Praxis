# AI 套话清单

write-prose 必须删掉的 AI 套话与 ABC 三件套。SKILL.md 的 Hard constraint 引用本文件。

## AI 套话

### 时间套话
- "在当今 AI 时代"
- "在快速发展的今天"
- "近年来 / 近些年来"
- "当前 / 目前"(开场白用)

### 进程套话
- "随着...的发展"
- "随着...的普及"
- "随着...的不断深入"

### 收尾套话
- "总之"
- "总而言之"
- "综上所述"
- "总的来说"

### 强调套话
- "值得注意的是"
- "需要指出的是"
- "重要的是"
- "特别地"

### 装深刻
- "深入探讨"
- "深入剖析"
- "深入挖掘"
- "深入思考"

### 假共识
- "众所周知"
- "我们都知道"
- "毋庸置疑"
- "不言而喻"

### 假权威
- "不可否认"
- "不容忽视"
- "客观来说"

### 模糊量词
- "在某种程度上"
- "在一定程度上"
- "某种意义上"
- "或多或少"

### 累加套话
- "不仅如此"
- "更重要的是"
- "更进一步"
- "另一方面"(用作过渡时)

## ABC 三件套(中文 LLM 最顽固模式)

逗号隔开 3 个并列形容词 / 动词。

### 例子(全部禁用)
- "高效、稳定、易用"
- "快速、清晰、可靠"
- "简洁、优雅、强大"
- "智能、便捷、安全"
- "深刻、细腻、动人"
- "深度、广度、温度"

### 改写方法
即使描述对象本身有 3 个属性,**分散到不同句子 / 段落**,不在一句逗号并列:

❌ "本工具高效、稳定、易用,适合所有开发者"

✓ "最直观的是它的速度。出问题的时候也不容易崩。新手用第一遍就上手 — 这点不太常见。"

### 反例(伪装的三件套也不行)
- 一句话三动词: "提升体验、降低门槛、加速增长" — 也是三件套
- 三动宾短语并列: "解决问题、提高效率、优化流程"
- 隐藏在排比里: "更快、更稳、更好"

## 写完 prose 之后必读的检查清单

1. **grep 套话**: 这清单的每个词,在产出里出现过就改
2. **段落开头/结尾**: 有没有套话句式("总的来说...""值得注意的是...")
3. **长句体检**: 每个长句,是不是 3 个并列形容词
4. **结构化**: H2 / H3 / bullet 数量,< 1000 字最多 1 层 H2
5. **句长节奏**: 短句长句交替,不全是中长句堆叠
6. **风格参考**: 用户给了风格(如"王小波")的话,产出真有那风格特征(短句 / 讥讽 / 口语),还是只在风格说明里 lip service?
